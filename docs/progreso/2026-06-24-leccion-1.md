# Progreso: del sensor al modelo

## Conceptos comprendidos

- Diferencia entre el identificador de una sala y el valor medido.
- Uso de `ifcGlobalId` para vincular telemetría con un objeto IFC.
- Funcionamiento y latencia del *polling* periódico.
- Reutilización de una lectura para color, paneles y alarmas.
- Importancia de la marca temporal y del estado del sensor.

## Modelo mental consolidado

```text
Sensor → plataforma IoT → controlador de entrada
       → fórmula → uno o varios controladores de representación
```

## Decisión importante

Un dato aparentemente seguro no debe mostrarse como válido cuando el sensor
está desconectado o la medición ha caducado. La interfaz debe distinguir entre:

- valor válido;
- dato obsoleto;
- sensor desconectado;
- error de lectura.
